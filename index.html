HTML redesigned for web apps!




Basics

Data Binding

HTML is an excellent templating language. AngularJS adds data-binding directly into HTML using the {{ expression }} notation. Form controls are bound bi-directionally and validated. This makes the model the single source of truth in your application.

Controller

Controller is the code behind the DOM. It is pure application behavior in an unpolluted imperative way. Define your behavior here, update the model on server or user interaction and you are done. No need to register callbacks, or manipulate the DOM, just watch the data-binding magic do the work.

Plain JavaScript

AngularJS wants to be unobtrusive. Unlike other frameworks, there is no need to inherit from proprietary types; to wrap the model in assessor methods; or to use proprietary collections and object hashes. Just plain old JavaScript here. This makes your code easy to test, maintain, reuse, and free from boilerplate.

Intermediate

Deep Linking

Angular apps are structured like classical round-trip apps with URLs, but unlike round-trip apps, Angular apps execute in the browser for instant navigation. The URL based focus means that deep linking works out of the box, and that there is no need to register click listeners on URLs. This is a development model which the web-developers are familiar with. An added benefit is that the application is composed from separate pages, which are loaded on as needed basis, resulting in smaller download and faster startup.

Form Validation

Form validation is an important part of great user experience. Angular provides declarative primitives to make it easy to build validation into your application in a declarative way. The validation declaration follows the spirit of HTML, and the result of validation is available for data-binding, giving you full control over the way user feedback is rendered.

Server Communication

XHR is great, but too low level. AngularJS wraps XHR to give you higher level features: data-binding integration, promises, central exception management. Even higher level abstraction called Resources is also available to hide all of the underlying details, and provide JavaScript objects for your server based resources.

Advanced

Directives

Directives is the most unique and powerful feature available only in Angular. Directives allow you to teach the browser new tricks! Directives let you add behavior to existing HTML constructs or to create brand new HTML constructs, such as tabs dialogs, etc... With directives you can turn HTML into declarative language which fits your needs.

Reusable Components

Turn often used HTML fragments into reusable directives to simplify your application layout.

Localization

Important part of serious apps is localization. Angular's local aware filters and stemming directives give you bulding blocks to make your application available in all languages.
